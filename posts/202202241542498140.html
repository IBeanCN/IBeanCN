mvnd确实提升了打包速度

发表于2022-02-24|更新于2022-06-24|Maven
|阅读量:|评论数:

长见识了

刚装了一下前段时间很火的mvnd，maven打包很爽，非常快，测试结果如下

mvnd package -Dmaven.test.skip=true

所有打包都在根目录下进行，并且进行了clean

原Maven

使用mvnd

另外如果不进行clean，使用mvnd

可见效果提升非常明显。

另外，idea自身也可以设置maven的线程数量，但是打包速度依旧赶不上使用mvnd。

准备开始

不添加任何防腐剂和啰嗦的东西，照着做绝对成~

准备工作

  1. 去github逛一逛，地址：https://github.com/apache/maven-mvnd
  2. 直接看手动安装，当然如果你们的Windows或者Mac安装管理器，那就可以直接关掉该文章，直接看原文档了

手动安装

  • 从https://github.com/mvndaemon/mvnd/releases下载安装的zip；

  • 解压到你想解压的位置
  • 把解压后位置中的bin添加到环境变量里；

  • 如果设置了JAVA_HOME的环境变量可以跳过这一步，没配置的可以在conf目录下找到 mvnd.properties，然后配置下java.home

  • 测试mvnd是否配置成功，重新打开一个新的黑框框，执行mvnd --version，如果弹出类似的信息则配置成功

    $ mvnd –version Maven Daemon 0.0.11-linux-amd64 (native) Terminal: org.jline.terminal.impl.PosixSysTerminal with pty org.jline.terminal.impl.jansi.osx.OsXNativePty Apache Maven 3.6.3 (cecedd343002696d0abb50b32b541b8a6ba2883f) Maven home: /home/ppalaga/orgs/mvnd/mvnd/daemon/target/maven-distro Java version: 11.0.1, vendor: AdoptOpenJDK, runtime: /home/data/jvm/adopt-openjdk/jdk-11.0.1+13 Default locale: en_IE, platform encoding: UTF-8 OS name: “linux”, version: “5.6.13-200.fc31.x86_64”, arch: “amd64”, family: “unix”

  • 如果Windows安装过程报错 VCRUNTIME140.dll was not found

    需要安装 vc_redist.x64.exe from https://support.microsoft.com/en-us/help/2977003/the-latest-supported-visual-c-downloads.

    See https://github.com/oracle/graal/issues/1762 for more information.

常用命令

mvnd --status：查看守护线程的状态

mvnd --stop：会停止所有mvnd的守护线程

mvnd的命令使用和maven是一样的
